### 1.3 Real-World User Stories

**Story A: Field Registration**
> Officer Thabo visits farmer Mamello at her farm in Berea District. While sitting under a tree, he registers her personal details on his tablet. When they reach the "Location & Land" section, Mamello walks Thabo around her maize field. At each corner, Thabo taps "Mark Corner" on his tablet. The boundary appears on screen as they walk. When complete, the system shows "0.82 hectares" â€” Mamello nods, that's about right.

**Story B: Office Digitization**
> Back at the district office, Officer Thabo has paper sketches from yesterday's visits to three farms. He opens each farmer's draft registration on his desktop computer. Using satellite imagery, he traces the field boundaries by clicking on the corners he recognizes from his visit. The system calculates areas automatically.

**Story C: Mixed Approach**
> For farmer Ntate Mohapi's large grazing land, Thabo captured only the four main corners using GPS during his visit. Back at the office, he refines the boundary by adjusting vertices on satellite imagery to follow the actual fence line visible in the image.

## 7. ACCESSIBILITY CONSIDERATIONS

### 7.1 Keyboard Navigation (Desktop)

| Key | Action |
|-----|--------|
| **Tab** | Move between interactive elements |
| **Enter** | Activate button, place vertex (when map focused) |
| **Escape** | Cancel current action, close dialogs |
| **Arrow keys** | Pan map (when map focused) |
| **+/-** | Zoom in/out (when map focused) |
| **Delete** | Remove selected vertex (edit mode) |
| **Ctrl+Z** | Undo last action |

### 7.2 Screen Reader Announcements

| Event | Announcement |
|-------|--------------|
| **Vertex placed** | "Corner 3 marked. 4 corners total. Area approximately 0.8 hectares." |
| **Polygon closed** | "Shape complete. 4 corners. 0.82 hectares. Press Tab to review options." |
| **Validation error** | "Warning: Polygon lines are crossing. Please adjust corners." |
| **Save success** | "Parcel 'Maize Field' saved successfully." |

### 7.3 Color Contrast

All text and interactive elements meet WCAG 2.1 AA standards (4.5:1 minimum contrast ratio).

## 8. SUMMARY: KEY DESIGN DECISIONS

| Decision | Rationale |
|----------|-----------|
| **Mode selection before drawing** | Clear mental model, appropriate tool for context |
| **Three-step wizard (Draw Mode)** | Progressive disclosure, reduces cognitive load |
| **Large GPS indicator (Walk Mode)** | Critical for field accuracy, must be glanceable |
| **Real-time area calculation** | Immediate feedback helps users verify boundaries |
| **Midpoint handles for editing** | Intuitive way to add vertices, common in drawing apps |
| **Summary view with map + list** | Overview of all parcels, easy navigation |
| **Overlap detection with visual** | Prevents data errors, clear remediation path |
| **Offline-first for Walk Mode** | Essential for rural areas with poor connectivity |
